# 5. Поды (Pods)

## Что такое Pod

**Pod** — это **минимальная единица развёртывания** в Kubernetes. Pod содержит один или несколько контейнеров, которые:

- Запускаются **на одной ноде**.
- Делят **сетевой namespace** (общий IP, общие порты, друг к другу — через `localhost`).
- Делят **тома** (volumes), если они объявлены в Pod.

99% случаев в одном Pod — **один контейнер**. Несколько контейнеров кладут вместе только если они тесно связаны (sidecar, прокси, лог-агент).

## Что важно понять про Pod

1. **Под — недолговечный**. Если нода падает или Pod падает «насмерть», его не воскрешают. Воскрешением занимаются контроллеры (ReplicaSet/Deployment).
2. **У Pod нет «постоянного» IP** в общем случае. После пересоздания IP другой.
3. **«Голый» Pod** (без контроллера) почти никогда не используют в проде. Обычно создают **Deployment**, а тот уже поднимает Pods.

## Типичные поля

- `spec.containers[]` — список контейнеров: `name`, `image`, `ports`, `env`, `resources`.
- `spec.restartPolicy` — `Always` (по умолчанию для Pod из Deployment), `OnFailure`, `Never`.
- `spec.nodeSelector` / `affinity` — куда планировать.
- `spec.volumes` — общие тома.

## Жизненный цикл (фазы)

| Фаза | Смысл |
|------|--------|
| **Pending** | Pod создан, но контейнеры ещё не запущены (ждёт ноду, образ скачивается). |
| **Running** | Хотя бы один контейнер запущен. |
| **Succeeded** | Все контейнеры завершились **успешно** и не перезапускаются. |
| **Failed** | Все контейнеры завершились, и хотя бы один — с ошибкой. |
| **Unknown** | Состояние нельзя определить. |

Дополнительно в `READY` показывается, прошёл ли **readinessProbe**: `1/1`, `0/1` и т.д.

## Полезные команды

```bash
kubectl get pods
kubectl get pods -o wide              # + IP и нода
kubectl describe pod <name>           # события и причина проблем
kubectl logs <name>                   # логи (если контейнер один)
kubectl logs <name> -c <container>    # логи конкретного контейнера
kubectl exec -it <name> -- sh         # войти в контейнер
kubectl delete pod <name>             # удалить pod
```

## Быстрый запуск пода (без YAML)

```bash
kubectl run nginx --image=nginx:1.27
kubectl get pods
kubectl delete pod nginx
```

`kubectl run` хорош для одноразовых проб; в реальной жизни Pod описывают **YAML-манифестом** и применяют через `kubectl apply -f`.
