# 3. Элементы Kubernetes

В Kubernetes всё, чем можно управлять, — это **объекты** (resources). Вы описываете **желаемое состояние** в YAML, а контроллеры приводят кластер к этому состоянию.

## Базовые объекты

| Объект | Зачем |
|--------|--------|
| **Pod** | Минимальная единица запуска: один или несколько контейнеров с общим сетевым пространством и томами. |
| **ReplicaSet** | Гарантирует, что в кластере работает **N одинаковых подов**. |
| **Deployment** | Управляет ReplicaSet'ами: обновления, откаты, масштабирование. На практике вы пишете именно Deployment, а ReplicaSet создаётся автоматически. |
| **Service** | Стабильный сетевой адрес и балансировка для группы подов (поды приходят и уходят, Service остаётся). |
| **Namespace** | Логическое разделение объектов внутри одного кластера (изоляция, квоты, RBAC). |

## Дальше в курсе встретятся

| Объект | Зачем (кратко) |
|--------|-----------------|
| **ConfigMap** | Конфигурация в виде ключ-значение (не секретная). |
| **Secret** | То же, но для паролей/токенов. |
| **PersistentVolume / PersistentVolumeClaim** | Постоянное хранилище для подов. |
| **Job / CronJob** | Разовые и периодические задачи. |
| **Ingress** | Внешний HTTP(S)-вход в кластер с маршрутизацией по хосту/пути. |
| **DaemonSet** | По одному поду на каждой (или выбранной) ноде. |
| **StatefulSet** | Для приложений с состоянием (БД, очереди), упорядоченное развёртывание. |

## Общая структура любого объекта

```yaml
apiVersion: <группа/версия API>
kind: <тип объекта>
metadata:
  name: <имя>
  namespace: <namespace, по умолчанию default>
  labels:
    <ключ>: <значение>
spec:
  # желаемое состояние, специфично для каждого kind
status:
  # фактическое состояние (заполняет сам кластер)
```

Вы редактируете **`spec`**, а **`status`** показывает, что получилось на самом деле.

## Как смотреть объекты

```bash
kubectl get pods
kubectl get deployments
kubectl get svc            # Service
kubectl get ns             # Namespace
kubectl api-resources      # все доступные виды объектов
```

Подробности по конкретному объекту:

```bash
kubectl describe pod <name>
kubectl get pod <name> -o yaml
```
